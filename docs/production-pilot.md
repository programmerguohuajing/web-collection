# Web Collection 生产试点清单

## 试点范围

- Web：选择一个核心交易或运营后台，接入错误、性能、行为和会话回放。
- 移动端：选择一个 uni-app、Taro 或 React Native 业务，接入统一用户信息和关键行为。
- 试点周期：建议 2 至 4 周，覆盖至少一次正式发版。

## 上线门禁

1. 为业务分配唯一 `appId`，登记负责人并设置事件与回放采样率。
2. SDK 初始化时设置 `release`，登录后调用 `setUser` 补充用户信息。
3. 构建流水线执行 `pnpm sourcemaps:upload`，确保 SourceMap 与 `appId/release` 一致。
4. 配置 `ADMIN_API_KEY`、`COLLECT_TOKEN`、PostgreSQL 备份和 `FEISHU_WEBHOOK_URL`。
5. 在采集治理页面设置保留周期、错误告警、回归告警和性能阈值。
6. 验证手机号展示脱敏、采集 URL 敏感参数清洗和管理接口鉴权。

## 验收指标

| 维度 | 验收标准 |
| --- | --- |
| 数据完整性 | 事件带有 appId、release、sessionId，登录用户带 userId |
| 错误定位 | 测试错误可聚合、可 SourceMap 反解、修复后可标记解决 |
| 错误回归 | 已解决错误在新版本重现时产生 regression 告警 |
| 性能监控 | LCP、INP、CLS、长任务、慢接口和慢资源可查询 |
| 会话回放 | 路由切换生成独立回放，抽样记录可正常播放 |
| 成本治理 | 采样率和保留周期生效，过期数据可自动清理 |
| 运营复盘 | 事件、错误和回放摘要可导出 CSV |

## 回滚方案

- 将应用的事件采样率和回放采样率设置为 `0`，无需业务重新发版即可停止入库。
- 紧急情况下移除 SDK 初始化或关闭采集域名流量。
- SourceMap 仅用于服务端反解，不对公网暴露源文件内容。
